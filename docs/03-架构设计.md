## 核心模块设计

### 1. API接口层

位于 `backend/api/`，负责处理HTTP请求和响应。

#### 主要模块

**main.py** - FastAPI主应用
- 应用初始化和配置
- 中间件配置（CORS、日志等）
- 路由注册
- 优雅关闭机制
- 健康检查接口

**upload.py** - 分片上传接口
- 分片上传：`POST /api/upload/chunk`
- 完成上传：`POST /api/upload/complete`
- 查询状态：`GET /api/upload/status/{uploadId}`

**unsupervised_detection.py** - 无监督检测接口
- 上传影像：`POST /api/unsupervised/upload-image`
- 启动检测：`POST /api/unsupervised/detect`
- 查询状态：`GET /api/unsupervised/task-status/{taskId}`
- 取消任务：`POST /api/unsupervised/cancel/{taskId}`

**training_sample.py** - 训练样本接口
- 样本上传、查询、删除等操作

**detection_config.py** - 检测配置接口
- 模型配置、参数设置等

**task_status.py** - 任务状态接口
- 统一的任务状态查询接口

### 2. 业务逻辑层

位于 `backend/services/`，实现核心业务逻辑。

#### 后台任务管理器 (background_task_manager.py)

核心功能：
- 任务生命周期管理
- 进度实时追踪
- 任务持久化存储
- 强制终止机制
- 子进程管理

### 3. 模型层

位于 `backend/models/`，定义数据模型和深度学习模型。

**feature_fusion.py** - 特征融合

**model_factory.py** - 模型工厂

**task_storage.py** - 任务存储

## 关键技术实现

### 1. 优雅关闭机制

实现位置：`backend/api/main.py`

```python
class GracefulShutdownManager:
    def __init__(self):
        self.shutdown_event = asyncio.Event()
        self.is_shutting_down = False

    async def shutdown(self):
        # 1. 停止接受新任务
        self.is_shutting_down = True

        # 2. 终止运行中的任务
        await task_manager.shutdown()

        # 3. 等待子进程退出
        await self._wait_for_processes()

        # 4. 清理资源
        await self._cleanup()
```
