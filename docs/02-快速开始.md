# 快速开始

## 环境准备

### 1. 系统要求

- **操作系统**：Linux、macOS或Windows
- **Python**：3.8及以上版本
- **Node.js**：16.x及以上版本
- **GDAL**：3.x及以上版本

### 2. 安装依赖

#### 安装GDAL

**macOS:**
```bash
brew install gdal
```

**Ubuntu/Debian:**
```bash
sudo apt-get update
sudo apt-get install gdal-bin libgdal-dev
```

**Windows:**
下载并安装OSGeo4W或使用conda安装

#### 安装Python依赖

```bash
# 创建虚拟环境（推荐）
python -m venv venv
source venv/bin/activate  # Linux/macOS
# 或
venv\Scripts\activate  # Windows

# 安装依赖
pip install -r requirements.txt
```

#### 安装前端依赖

```bash
cd frontend
npm install
```

## 配置说明

### 1. 后端配置

配置文件位于 `backend/config/settings.py`

主要配置项：

```python
# 存储目录配置
STORAGE_DIR = PROJECT_ROOT / "storage"
UPLOAD_DIR = STORAGE_DIR / "uploads"
TRAINING_SAMPLES_DIR = STORAGE_DIR / "training_samples"
DETECTION_IMAGES_DIR = STORAGE_DIR / "detection_images"

# 影像处理配置
IMAGE_PROCESSING_CONFIG = {
    "chunk_width": 1024,        # 瓦片宽度
    "chunk_height": 1024,       # 瓦片高度
    "parallel_chunks": 8,       # 并行处理数量
}

# 数值库线程限制
NUMERICAL_LIBRARY_THREADS = 2  # 每个worker使用的线程数
```

### 2. 创建必要目录

系统会自动创建所需目录，也可以手动创建：

```bash
mkdir -p storage/{uploads,training_samples,detection_images,temp,models}
mkdir -p logs
mkdir -p data
```

## 启动服务

### 方式一：使用启动脚本（推荐）

```bash
# 赋予执行权限
chmod +x start.sh

# 启动服务
./start.sh
```

启动脚本会自动：
1. 检查Python环境
2. 安装/更新依赖
3. 启动后端服务（端口8000）
4. 启动前端服务（端口5173）

### 方式二：手动启动

#### 启动后端

```bash
cd backend
uvicorn api.main:app --host 0.0.0.0 --port 8000 --reload
```

#### 启动前端

```bash
cd frontend
npm run dev
```

## 访问系统

启动成功后，可以通过以下地址访问：

- **前端界面**：http://localhost:5173
- **后端API**：http://localhost:8000
- **API文档**：http://localhost:8000/docs
- **健康检查**：http://localhost:8000/health

## 快速体验

### 1. 无监督检测（无需训练）

无监督检测不需要训练样本，可以直接使用：

1. 访问前端界面
2. 点击"无监督检测"菜单
3. 上传待检测的遥感影像（支持.tif、.img格式）
4. 设置检测参数：
   - 聚类数量（默认5）
   - 最小面积（默认100平方米）
5. 点击"开始检测"
6. 等待检测完成，查看结果

### 2. 监督检测（需要训练）

监督检测需要先训练模型：

#### 步骤1：准备训练样本

1. 点击"训练样本管理"菜单
2. 上传训练样本影像
3. 等待样本上传完成

#### 步骤2：训练模型

1. 点击"模型训练"菜单
2. 选择训练样本
3. 配置模型参数：
   - 主干网络（VGG或ResNet）
   - 训练轮数
   - 学习率
   - 批次大小
4. 点击"开始训练"
5. 等待训练完成

#### 步骤3：执行检测

1. 点击"监督检测"菜单
2. 选择训练好的模型
3. 上传待检测影像
4. 点击"开始检测"
5. 查看检测结果

## 常见问题

### 1. 端口被占用

如果8000或5173端口被占用，可以修改端口：

**后端：**
```bash
uvicorn api.main:app --host 0.0.0.0 --port 8001
```

**前端：**
修改 `frontend/vite.config.ts` 中的端口配置

### 2. GDAL导入错误

确保GDAL已正确安装，并且Python能找到GDAL库：

```bash
# 测试GDAL
python -c "from osgeo import gdal; print(gdal.__version__)"
```

### 3. 内存不足

如果处理大影像时内存不足，可以：
- 减少并行处理数量（修改 `parallel_chunks`）
- 减少数值库线程数（修改 `NUMERICAL_LIBRARY_THREADS`）
- 增加系统内存

### 4. 上传失败

检查：
- 文件格式是否支持（.tif、.img）
- 文件大小是否超过限制
- 存储目录是否有写权限
- 网络连接是否稳定

## 验证安装

使用启动脚本的验证功能：

```bash
# 运行验证
./start.sh --verify

# 运行基础测试
./start.sh --test

# 运行高级测试
./start.sh --test-advanced
```

## 查看日志

日志文件位于 `logs/` 目录：

```bash
# 查看最新日志
./start.sh --check-logs

# 或手动查看
tail -f logs/app.log
```

## 停止服务

按 `Ctrl+C` 停止服务，系统会优雅关闭：
1. 停止接受新任务
2. 终止运行中的任务
3. 等待子进程退出
4. 清理资源
