# 部署运维

## 生产环境部署

### 1. 服务器要求

#### 硬件配置

**最低配置**:
- CPU: 4核
- 内存: 16GB
- 硬盘: 100GB SSD
- 网络: 100Mbps

**推荐配置**:
- CPU: 8核或以上
- 内存: 32GB或以上
- 硬盘: 500GB SSD
- 网络: 1Gbps
- GPU: NVIDIA GPU（可选，用于加速）

#### 软件环境

- 操作系统: Ubuntu 20.04 LTS或以上
- Python: 3.8+
- Node.js: 16.x+
- GDAL: 3.x+
- Nginx: 1.18+（可选，用于反向代理）

### 2. 系统依赖安装

```bash
# 更新系统
sudo apt-get update
sudo apt-get upgrade -y

# 安装Python和pip
sudo apt-get install python3 python3-pip python3-venv -y

# 安装GDAL
sudo apt-get install gdal-bin libgdal-dev -y

# 安装Node.js
curl -fsSL https://deb.nodesource.com/setup_16.x | sudo -E bash -
sudo apt-get install nodejs -y

# 安装Nginx（可选）
sudo apt-get install nginx -y

# 安装其他依赖
sudo apt-get install git build-essential -y
```

### 3. 部署后端

#### 创建部署目录

```bash
sudo mkdir -p /opt/disease-detection
sudo chown $USER:$USER /opt/disease-detection
cd /opt/disease-detection
```

#### 克隆代码

```bash
git clone <repository-url> .
```

#### 配置Python环境

```bash
# 创建虚拟环境
python3 -m venv venv
source venv/bin/activate

# 安装依赖
pip install -r requirements.txt

# 安装生产环境依赖
pip install gunicorn
```

#### 配置环境变量

创建 `.env` 文件：
```bash
# 应用配置
APP_ENV=production
DEBUG=False

# 存储配置
STORAGE_DIR=/opt/disease-detection/storage

# 数据库配置（如果使用）
DATABASE_URL=sqlite:///./disease_detection.db

# 日志配置
LOG_LEVEL=INFO
LOG_DIR=/var/log/disease-detection
```

#### 创建systemd服务

创建 `/etc/systemd/system/disease-detection.service`:
```ini
[Unit]
Description=Disease Detection Backend Service
After=network.target

[Service]
Type=notify
User=www-data
Group=www-data
WorkingDirectory=/opt/disease-detection
Environment="PATH=/opt/disease-detection/venv/bin"
ExecStart=/opt/disease-detection/venv/bin/uvicorn backend.api.main:app --host 0.0.0.0 --port 8000 --workers 4
Restart=always
RestartSec=10

[Install]
WantedBy=multi-user.target
```

启动服务：
```bash
sudo systemctl daemon-reload
sudo systemctl enable disease-detection
sudo systemctl start disease-detection
sudo systemctl status disease-detection
```

### 4. 部署前端

#### 构建前端

```bash
cd frontend
npm install
npm run build
```

#### 配置Nginx

创建 `/etc/nginx/sites-available/disease-detection`:
```nginx
server {
    listen 80;
    server_name your-domain.com;

    # 前端静态文件
    location / {
        root /opt/disease-detection/frontend/dist;
        try_files $uri $uri/ /index.html;
    }

    # 后端API代理
    location /api {
        proxy_pass http://localhost:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;

        # 上传文件大小限制
        client_max_body_size 10G;
        proxy_request_buffering off;
    }

    # WebSocket支持（如果需要）
    location /ws {
        proxy_pass http://localhost:8000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
    }
}
```

启用站点：
```bash
sudo ln -s /etc/nginx/sites-available/disease-detection /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl reload nginx
```

## Docker部署

### 1. 创建Dockerfile

**后端Dockerfile** (`Dockerfile.backend`):
```dockerfile
FROM python:3.9-slim

# 安装系统依赖
RUN apt-get update && apt-get install -y \
    gdal-bin \
    libgdal-dev \
    build-essential \
    && rm -rf /var/lib/apt/lists/*

# 设置工作目录
WORKDIR /app

# 复制依赖文件
COPY requirements.txt .

# 安装Python依赖
RUN pip install --no-cache-dir -r requirements.txt

# 复制代码
COPY backend/ ./backend/
COPY storage/ ./storage/

# 暴露端口
EXPOSE 8000

# 启动命令
CMD ["uvicorn", "backend.api.main:app", "--host", "0.0.0.0", "--port", "8000"]
```

**前端Dockerfile** (`Dockerfile.frontend`):
```dockerfile
FROM node:16 AS builder

WORKDIR /app
COPY frontend/package*.json ./
RUN npm install
COPY frontend/ .
RUN npm run build

FROM nginx:alpine
COPY --from=builder /app/dist /usr/share/nginx/html
COPY nginx.conf /etc/nginx/conf.d/default.conf
EXPOSE 80
CMD ["nginx", "-g", "daemon off;"]
```

### 2. 创建docker-compose.yml

```yaml
version: '3.8'

services:
  backend:
    build:
      context: .
      dockerfile: Dockerfile.backend
    ports:
      - "8000:8000"
    volumes:
      - ./storage:/app/storage
      - ./logs:/app/logs
    environment:
      - APP_ENV=production
      - DEBUG=False
    restart: always

  frontend:
    build:
      context: .
      dockerfile: Dockerfile.frontend
    ports:
      - "80:80"
    depends_on:
      - backend
    restart: always

volumes:
  storage:
  logs:
```

### 3. 启动服务

```bash
# 构建镜像
docker-compose build

# 启动服务
docker-compose up -d

# 查看日志
docker-compose logs -f

# 停止服务
docker-compose down
```

## 监控和日志

### 1. 日志管理

#### 日志配置

后端日志配置在 `backend/utils/logger.py`:
```python
import logging
from logging.handlers import RotatingFileHandler

def setup_logger():
    logger = logging.getLogger("disease_detection")
    logger.setLevel(logging.INFO)

    # 文件处理器（自动轮转）
    file_handler = RotatingFileHandler(
        "logs/app.log",
        maxBytes=10*1024*1024,  # 10MB
        backupCount=10
    )
    file_handler.setFormatter(
        logging.Formatter(
            '%(asctime)s - %(name)s - %(levelname)s - %(message)s'
        )
    )
    logger.addHandler(file_handler)

    return logger
```

#### 查看日志

```bash

# 查看最近100行
tail -n 100 logs/app.log

# 搜索错误日志

# 查看特定时间段的日志

```

## 备份和恢复

### 1. 数据备份

#### 备份脚本

创建 `scripts/backup.sh`:
```bash
#!/bin/bash

BACKUP_DIR="/backup/disease-detection"
DATE=$(date +%Y%m%d_%H%M%S)

# 创建备份目录
mkdir -p $BACKUP_DIR

# 备份数据库
cp /opt/disease-detection/disease_detection.db $BACKUP_DIR/db_$DATE.db

# 备份存储目录
tar -czf $BACKUP_DIR/storage_$DATE.tar.gz /opt/disease-detection/storage

# 备份配置文件
cp /opt/disease-detection/.env $BACKUP_DIR/env_$DATE

# 删除30天前的备份
find $BACKUP_DIR -name "*.tar.gz" -mtime +30 -delete
find $BACKUP_DIR -name "*.db" -mtime +30 -delete

echo "备份完成: $DATE"
```

#### 定时备份

添加到crontab：
```bash
# 编辑crontab
crontab -e

# 每天凌晨2点备份
0 2 * * * /opt/disease-detection/scripts/backup.sh >> /var/log/backup.log 2>&1
```

### 2. 数据恢复

```bash
# 恢复数据库
cp /backup/disease-detection/db_20240101_020000.db /opt/disease-detection/disease_detection.db

# 恢复存储目录
tar -xzf /backup/disease-detection/storage_20240101_020000.tar.gz -C /

# 重启服务
sudo systemctl restart disease-detection
```

## 故障排查

### 1. 服务无法启动

#### 检查日志
```bash
sudo journalctl -u disease-detection -n 50
```

#### 常见原因
- 端口被占用
- 依赖缺失
- 配置错误
- 权限问题

#### 解决方法
```bash
# 检查端口
sudo lsof -i :8000

# 检查依赖
source venv/bin/activate
pip list

# 检查配置
cat .env

# 检查权限
ls -la /opt/disease-detection
```

### 2. 内存不足

#### 症状
- 进程被杀死（OOM Killer）
- 系统响应缓慢
- 日志中出现内存错误

#### 解决方法
```bash
# 查看内存使用
free -h
top

# 减少worker数量
# 修改systemd服务文件，减少--workers参数

# 增加swap空间
sudo fallocate -l 4G /swapfile
sudo chmod 600 /swapfile
sudo mkswap /swapfile
sudo swapon /swapfile
```
