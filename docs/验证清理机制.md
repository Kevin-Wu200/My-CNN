# 验证任务文件清理机制

## 验证步骤

### 1. 启动服务
```bash
cd /Users/wuchenkai/深度学习模型
./start.sh
```

### 2. 上传测试影像
使用前端或API上传一个测试影像文件，记录：
- uploadId
- 文件名

### 3. 检查文件存在
```bash
# 检查 merged 文件
ls -lh storage/merged/

# 检查 detection_images 文件
ls -lh storage/detection_images/
```

### 4. 启动检测任务
通过API启动无监督检测任务，记录 task_id

### 5. 等待任务完成
查询任务状态，确认任务完成

### 6. 观察日志
查看日志中的清理调度信息：
```bash
tail -f logs/app.log | grep CLEANUP
```

应该看到：
- `[CLEANUP_SCHEDULED]` - 清理已调度
- `[CLEANUP_THREAD_STARTED]` - 清理线程已启动

### 7. 等待120秒后检查
```bash
# 等待120秒后，检查文件是否被删除
ls -lh storage/merged/
ls -lh storage/detection_images/
```

### 8. 验证检测结果仍然存在
通过API查询任务结果，确认检测结果仍在数据库中：
```bash
curl http://localhost:8000/api/tasks/{task_id}
```

## 预期结果

✅ 任务完成后，相关文件在120秒后被删除
✅ 检测结果仍然可以通过API查询
✅ 日志中有完整的清理记录
✅ 没有错误或异常

## 日志关键字

搜索以下日志关键字来追踪清理过程：
- `CLEANUP_SCHEDULED` - 清理已调度
- `CLEANUP_THREAD_STARTED` - 清理线程启动
- `CLEANUP_EXECUTING` - 开始执行清理
- `CLEANUP_START` - 清理开始
- `CLEANUP_FILE_FOUND` - 找到需要清理的文件
- `CLEANUP_DELETING` - 正在删除文件
- `FILE_DELETED` - 文件已删除
- `CLEANUP_COMPLETE` - 清理完成

## 故障排查

如果清理没有执行：
1. 检查日志中是否有 `CLEANUP_SCHEDULED`
2. 检查是否有错误日志
3. 确认文件路径是否正确
4. 确认文件是否在允许删除的目录中
