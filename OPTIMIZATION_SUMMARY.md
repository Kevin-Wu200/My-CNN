# 后端影像处理优化总结

## 优化目标
在不改变现有功能的前提下，对后端影像处理进行以下优化：
- **分块大小**：1024×1024 像素
- **并行处理数量**：8 个分块

## 优化内容

### 1. 并行处理服务优化 (`backend/services/parallel_processing.py`)

#### 变更内容：
- 添加常量 `DEFAULT_PARALLEL_WORKERS = 8`，定义默认并行处理数量
- 优化 `get_auto_worker_count()` 方法，优先使用 8 个工作进程
- 更新日志信息，明确显示使用的并行处理数量

#### 优化逻辑：
```python
# 优先使用默认的 8 个工作进程
num_workers = min(DEFAULT_PARALLEL_WORKERS, cpu_count, max_limit)
```

**优势**：
- 确保在大多数系统上都能使用 8 个工作进程
- 如果 CPU 核心数不足，自动降级到可用的最大值
- 保持向后兼容性

### 2. 深度学习检测服务优化 (`backend/services/detection.py`)

#### 变更内容：
- `detect_on_tiled_image()` 方法的 `num_workers` 参数默认值从 `None` 改为 `8`
- 更新文档说明，明确指出默认使用 8 个工作进程

#### 影响范围：
- 病害木检测任务自动使用 8 个工作进程进行并行处理
- 1024×1024 分块自动并行处理

### 3. 无监督检测服务优化 (`backend/services/unsupervised_detection.py`)

#### 变更内容：
- `detect_on_tiled_image()` 方法的 `num_workers` 参数默认值从 `None` 改为 `8`
- 更新文档说明，明确指出默认使用 8 个工作进程

#### 影响范围：
- 无监督病害木检测任务自动使用 8 个工作进程进行并行处理
- 1024×1024 分块自动并行处理

### 4. 配置文件更新 (`backend/config/settings.py`)

#### 变更内容：
- 更新 `IMAGE_PROCESSING_CONFIG` 中的注释，明确说明并行处理优化配置
- 确保 `parallel_chunks` 配置值为 8

## 分块处理流程

### 当前架构
```
用户上传影像
    ↓
后端接收并保存
    ↓
生成 1024×1024 分块
    ↓
并行处理 8 个分块（深度学习检测或无监督分类）
    ↓
合并检测结果
    ↓
返回检测结果给前端
```

### 优化后的处理流程

#### 深度学习检测流程
1. **分块阶段**：使用 `TilingService.generate_tiles()` 将影像分块为 1024×1024
2. **并行处理阶段**：使用 `ParallelProcessingService.process_tiles_parallel()` 并行处理 8 个分块
   - 每个分块执行 SLIC 超像素分割
   - 对每个超像素进行模型推理
   - 提取检测坐标
3. **结果合并阶段**：合并所有分块的检测结果到原始影像坐标系

#### 无监督分类检测流程
1. **分块阶段**：使用 `TilingService.generate_tiles()` 将影像分块为 1024×1024
2. **并行处理阶段**：使用 `ParallelProcessingService.process_tiles_parallel()` 并行处理 8 个分块
   - 每个分块执行影像归一化
   - 构建光谱和纹理特征
   - 执行 K-means 聚类
   - 识别病害木候选类别
   - 空间后处理
3. **结果合并阶段**：合并所有分块的检测结果到原始影像坐标系

## 性能优化效果

### 并行处理优势
- **吞吐量提升**：8 个分块并行处理，相比顺序处理可显著提升处理速度
- **资源利用**：充分利用多核 CPU，提高系统资源利用率
- **可扩展性**：支持大尺寸影像的高效处理

### 分块大小选择
- **1024×1024**：平衡内存占用和处理效率
  - 足够大以减少分块数量
  - 足够小以保持内存占用在合理范围内
  - 适合 GPU 显存限制

## 兼容性说明

### 向后兼容性
- 所有变更都是参数默认值的调整，不改变 API 签名
- 现有代码无需修改即可使用新的优化配置
- 用户仍可通过显式传参覆盖默认值

### 配置灵活性
- 如需调整并行处理数量，可修改 `DEFAULT_PARALLEL_WORKERS` 常量
- 如需调整分块大小，可修改 `DEFAULT_TILE_SIZE` 常量（在 `tile_utils.py` 中）
- 所有参数都可在运行时通过函数参数进行覆盖

## 注意事项

1. **内存占用**：8 个并行分块会占用更多内存，确保系统有足够的可用内存
2. **CPU 核心数**：如果 CPU 核心数少于 8，系统会自动降级到可用的最大值
3. **GPU 处理**：如果使用 GPU 进行推理，需要确保 GPU 显存足够支持 8 个并行分块
