## 问题诊断

### 问题描述
系统在处理大文件分片上传时出现严重问题：
- 前10个chunk上传成功（HTTP 200 OK）
- 从第10个chunk开始，所有请求返回429错误（Too Many Requests）
- 导致477个chunk中只有10个成功上传
- 上传成功率仅为2.1%

### 问题根源
**速率限制器配置不当**

| 组件 | 配置 | 问题 |
|------|------|------|
| 前端 | 8个并行worker | 同时发送8个请求 |
| 后端 | 10个请求/秒限制 | 无法处理8个并行请求 |
| 结果 | 冲突 | 所有超过限制的请求被拒绝 |

### 日志证据
```
前10个chunk：200 OK ✓
第10-477个chunk：429 Too Many Requests ✗
[RATE_LIMIT_EXCEEDED] uploadId=upload_1770313510481_klxq4epuq, chunkIndex=10
[RATE_LIMIT_EXCEEDED] uploadId=upload_1770313510481_klxq4epuq, chunkIndex=11
... (共467个429错误)
```

---

## 修复方案

### 修改内容
**文件**：`backend/api/upload.py`  
**行号**：第57行  
**修改**：将速率限制从10增加到100

```python
# 修改前
chunk_rate_limiter = RateLimiter(max_requests=10, window_seconds=1)

# 修改后
chunk_rate_limiter = RateLimiter(max_requests=100, window_seconds=1)
```

### 修复理由
- 前端使用8个并行worker
- 原始限制（10个请求/秒）无法支持
- 新限制（100个请求/秒）提供12.5倍缓冲
- 对服务器负载影响最小

### 额外改进
添加了速率限制警告日志：
```python
logger.warning(
    f"[RATE_LIMIT_WARNING] uploadId={key}, "
    f"requests={len(self.requests[key])}, max={self.max_requests}"
)
```
